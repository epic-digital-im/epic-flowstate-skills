# Pre-Flight Hierarchy Check

**Purpose:** Mandatory check before creating any FlowState task. Verifies every parent entity exists and creates missing ones in order.
**Used by:** task-creation, flowstate-task-schema, flowstate-task-execution

---

## Absolute Rule: IDs Are Never Fabricated

All FlowState IDs are auto-generated by MCP tools. Agents NEVER manually construct, guess, or infer IDs. Read `.flowstate/config.json` for `orgId`, `workspaceId`, `codebaseId`, `projectId`. All other IDs come from MCP query/create responses. If an ID is unavailable, ask the user.

---

## Overview

Every FlowState task requires a valid parent chain. Before creating a task, walk up the chain and verify (or create) each level. Never skip this check — orphaned tasks with invalid parent IDs cannot be tracked or updated.

```
org_ → work_ → proj_ → mile_ → task_
 ↑       ↑       ↑       ↑       ↑
verify  verify  verify  verify  CREATE
```

---

## The Check (5 Steps)

### Step 1: Read Config

```bash
cat .flowstate/config.json           # root config
cat packages/<pkg>/.flowstate/config.json  # package config (monorepo)
```

Extract: `orgId`, `workspaceId`, `codebaseId`, `projectId`

### Step 2: Verify Org

```
collection-get orgs <orgId>
→ 404: invalid orgId — check config, ask user. DO NOT PROCEED.
```

Org errors are not auto-fixable. The user must supply the correct `orgId`.

### Step 3: Verify Workspace

```
collection-get workspaces <workspaceId>
→ 404: create workspace (flowstate-workspace-registration)
        write workspaceId to root .flowstate/config.json
        commit config
```

### Step 4: Verify Project

```
collection-get projects <projectId>
→ 404 + projectId in config: create with same ID (preserves milestone/task refs)
    collection-create projects { id: "<projectId>", name, title, orgId, workspaceId }
→ 404 + no projectId in config: create new project, write projectId to package config
→ Success: proceed
```

Always commit the package config before creating tasks.

### Step 5: Verify or Create Milestone

```
collection-query milestones { projectId: "<projectId>", name: "<group-name>" }
→ found: use its milestoneId
→ empty: create milestone:
    collection-create milestones {
      name: "<group>", title: "<Milestone Title>",
      projectId, goalId: "", status: "To Do",
      completed: false, archived: false, version: 1,
      orgId, workspaceId
    }
```

---

## Quick Reference

| Step | Entity    | If Missing                        | Config to Update                 |
| ---- | --------- | --------------------------------- | -------------------------------- |
| 1    | Config    | Run workspace-registration        | —                                |
| 2    | Org       | Ask user — not auto-fixable       | —                                |
| 3    | Workspace | Create + write to root config     | Root `.flowstate/config.json`    |
| 4    | Project   | Create (preserve ID if in config) | Package `.flowstate/config.json` |
| 5    | Milestone | Create with `goalId: ""`          | Record milestoneId for task use  |

---

## Common Mistakes

| Mistake                                            | Fix                                                        |
| -------------------------------------------------- | ---------------------------------------------------------- |
| Skipping the check when projectId exists in config | Run anyway — project may have been deleted                 |
| Creating project with NEW ID when projectId exists | Always use the existing ID to preserve child entity refs   |
| Using root config's projectId for package tasks    | Package configs are the authoritative source for projectId |
| Creating milestone with null goalId                | Use `""` (empty string) — schema requires goalId           |
