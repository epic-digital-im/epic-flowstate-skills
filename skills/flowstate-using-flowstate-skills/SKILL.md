# FlowState Skills Index

All FlowState skills for entity management, registration, workflow execution, and process automation. Each skill has YAML frontmatter with `name` and `description`.

## How to Use

**Invoke relevant skills BEFORE any response or action.** Use the `Skill` tool with the skill name prefixed by `flowstate-skills:` (e.g., `flowstate-skills:flowstate-task-schema`).

If you need to create a FlowState entity, check a schema, or run a workflow, invoke the matching skill first.

---

## Absolute Rule: IDs Are Never Fabricated

**It is NEVER acceptable to manually create, guess, infer, or recall a FlowState ID.** This is a global rule that applies to ALL entities in FlowState without exception: orgs, workspaces, codebases, projects, milestones, tasks, goals, documents, records, products, business plans, discussions, approvals, and every other collection.

Every FlowState ID is auto-generated by MCP tools when an entity is created. Agents do not construct IDs.

IDs come from exactly three sources:
1. **Config files** (`.flowstate/config.json`) for `orgId`, `workspaceId`, `codebaseId`, `projectId`
2. **MCP tool responses** (returned when creating or querying entities)
3. **The user** (when asked explicitly because the ID cannot be found)

If an ID is unavailable from sources 1 or 2, ask the user. Never proceed with a made-up value.

## Mandatory: Read Config Before Any MCP Call

Before making any MCP `collection-*`, `document-*`, `bizplan-*`, or `product-*` call, you MUST read the config file:

```
Read .flowstate/config.json -> extract orgId, workspaceId, codebaseId
```

| ID | Source | Never |
|----|--------|-------|
| `orgId` | Root `.flowstate/config.json` | Guess from project name |
| `workspaceId` | Root `.flowstate/config.json` | Use placeholder values |
| `codebaseId` | Root `.flowstate/config.json` | Recall from memory |
| `projectId` | Package `.flowstate/config.json` | Derive from package name |
| `milestoneId` | MCP query by projectId | Fabricate |
| `taskId` | MCP query by milestoneId | Fabricate |

This applies to every workflow skill, every schema skill, and every process.
