# FlowState Config Process

**Status:** Active
**Collection:** N/A (local file)
**File:** `.flowstate/config.json`
**Purpose:** Define, validate, and initialize the `.flowstate/config.json` file that provides FlowState context IDs to all processes running in a project folder
**Trigger:** Any process that needs `orgId`, `workspaceId`, `codebaseId`, or `projectId`

---

## Absolute Rule: IDs Are Never Fabricated

All FlowState IDs are auto-generated by MCP tools. Agents NEVER manually construct, guess, or infer IDs. Every ID must come from a config file, an MCP tool response, or the user. No exceptions, no entity types exempt.

---

## Overview

The `.flowstate/config.json` file is the **source of truth** for FlowState context IDs in a project folder. Every process — task execution, brainstorming, project audits, document creation — reads this file to resolve `orgId`, `workspaceId`, `codebaseId`, and `projectId`.

**A process cannot run correctly without a valid config.** If any required ID is missing or points to a deleted record, all child entities (projects, milestones, tasks) will be broken or unlinked.

### Config File Locations

There are two types of config files:

| Location                                     | Type           | Contains                             |
| -------------------------------------------- | -------------- | ------------------------------------ |
| `<repo-root>/.flowstate/config.json`         | Root config    | `orgId`, `workspaceId`, `codebaseId` |
| `packages/<pkg>/.flowstate/config.json`      | Package config | Same as root + `projectId`           |
| `packages/apps/<app>/.flowstate/config.json` | Package config | Same as root + `projectId`           |

The **root config** is the anchor for the entire repository. All package configs inherit from it and add `projectId`.

---

## Root Config: Required Fields

The root `.flowstate/config.json` **must** contain these three IDs:

```json
{
  "orgId": "org_XXXXX",
  "workspaceId": "work_XXXXX",
  "codebaseId": "code_XXXXX"
}
```

Without all three, no entity in the repository can be properly linked.

### Full Root Config Structure

```json
{
  "version": "1.0.0",
  "projectName": "<repo-name>",
  "projectType": "monorepo",
  "orgId": "<org_XXXXX>",
  "workspaceId": "<work_XXXXX>",
  "codebaseId": "<code_XXXXX>",
  "vault": {
    "name": "<vault-name>",
    "type": "1password"
  },
  "repository": {
    "url": "<https://github.com/org/repo>",
    "provider": "github"
  },
  "dependencies": {
    "node": "<node-version>",
    "packageManager": "<yarn|npm|pnpm>",
    "docker": true
  },
  "createdAt": "<ISO-8601-timestamp>",
  "lastUpdated": "<ISO-8601-timestamp>"
}
```

### Optional Root Config Fields

Large projects may also include product and goal linkage:

```json
{
  "products": [{ "id": "prod_XXXXX", "name": "<product-name>", "status": "<in-progress|planned>" }],
  "goals": [{ "id": "goal_XXXXX", "name": "<goal-name>" }]
}
```

---

## Package Config: Required Fields

Every package in a monorepo that is tracked in FlowState must have:

```json
{
  "orgId": "<org_XXXXX>",
  "workspaceId": "<work_XXXXX>",
  "codebaseId": "<code_XXXXX>",
  "projectId": "<proj_XXXXX>"
}
```

### Full Package Config Structure

```json
{
  "version": "1.0.0",
  "projectName": "@scope/package-name",
  "projectType": "<library|cli|fullstack|worker|app>",
  "projectId": "<proj_XXXXX>",
  "codebaseId": "<code_XXXXX>",
  "orgId": "<org_XXXXX>",
  "workspaceId": "<work_XXXXX>",
  "repository": {
    "url": "<https://github.com/org/repo>",
    "provider": "github"
  },
  "createdAt": "<ISO-8601-timestamp>",
  "lastUpdated": "<ISO-8601-timestamp>"
}
```

Package configs do **not** include `vault`, `dependencies`, or `products` — those are root-level concerns.

## Validation Process

Run this check before executing any process in a project folder:

### Step 1: Confirm Config Exists

```bash
test -f .flowstate/config.json && echo "exists" || echo "MISSING"
```

If missing → run [workspace-registration-process.md](./workspace-registration-process.md).

### Step 2: Read and Validate Root Config

```bash
cat .flowstate/config.json
```

Check for presence of all three required IDs:

| Field         | Required | Missing Action                                             |
| ------------- | -------- | ---------------------------------------------------------- |
| `orgId`       | Yes      | Ask user for org ID — see [org-schema.md](./org-schema.md) |
| `workspaceId` | Yes      | Run workspace-registration-process                         |
| `codebaseId`  | Yes      | Run codebase-registration-process                          |

### Step 3: Verify Each ID Exists in FlowState

```
collection-get orgs <orgId>
→ if 404: org deleted — ask user which org to use

collection-get workspaces <workspaceId>
→ if 404: workspace deleted — re-register (workspace-registration-process.md)

collection-get codebases <codebaseId>
→ if 404: codebase deleted — re-register (codebase-registration-process.md)
```

### Step 4: Validate Package Config (Monorepos)

For each package you are working in:

```bash
cat packages/<pkg>/.flowstate/config.json
```

| Field         | Required | Missing Action                                                       |
| ------------- | -------- | -------------------------------------------------------------------- |
| `projectId`   | Yes      | Create project record — see [project-schema.md](./project-schema.md) |
| `orgId`       | Yes      | Copy from root config                                                |
| `workspaceId` | Yes      | Copy from root config                                                |
| `codebaseId`  | Yes      | Copy from root config                                                |

```
collection-get projects <projectId>
→ if 404: project deleted — create with same ID (project-schema.md)
```

## Config Initialization

### New Repository (No Config)

Run the full registration flow:

```
1. workspace-registration-process.md
   → creates workspace + codebase
   → writes root .flowstate/config.json with orgId, workspaceId, codebaseId

2. workspace-codebase-audit-process.md (monorepos only)
   → creates per-package project records
   → writes package .flowstate/config.json files with projectId
```

### Existing Repo with Incomplete Config

If the root config exists but is missing fields:

| Missing Field | Action                                                                    |
| ------------- | ------------------------------------------------------------------------- |
| `orgId`       | Ask user; never guess. See [org-schema.md](./org-schema.md)               |
| `workspaceId` | Create workspace via workspace-registration-process.md                    |
| `codebaseId`  | Create codebase via codebase-registration-process.md                      |
| `projectId`   | Package config only — create project via workspace-codebase-audit-process |

### Restoring Deleted Records

If a config has valid-looking IDs but the records are gone from FlowState:

```
1. Attempt collection-get for each ID
2. For any 404:
   a. Create the record with the SAME ID from config
      (preserves all existing child entity references)
   b. Do NOT generate a new ID
3. Verify child entities (projects, milestones, tasks) still reference correct parent IDs
```

## ID Inheritance in Monorepos

All packages in a monorepo share the same `orgId`, `workspaceId`, and `codebaseId` from the root config. Only `projectId` differs per package.

```
Root config:     orgId ─────────────────────────────────────────┐
                 workspaceId ───────────────────────────────────┤ shared
                 codebaseId ────────────────────────────────────┤
                                                                │
pkg/flowstate-cli:  projectId: proj_HpfQyaJBl5  ───────────────┤ unique per package
pkg/flowstate-mcp:  projectId: proj_ubTez4-mxO  ───────────────┤
pkg/flowstate-workers: projectId: proj_J8jVdRouBv ─────────────┘
```

When writing package configs, always copy `orgId`, `workspaceId`, and `codebaseId` from the root config — never re-type them.

## Config File Conventions

| Rule                                   | Rationale                                           |
| -------------------------------------- | --------------------------------------------------- |
| Commit `.flowstate/config.json` to git | It is not a secret — it is a project pointer        |
| Never put secrets in config            | Use 1Password vault references only                 |
| Never use placeholder IDs              | `"orgId": "default"` breaks all child entities      |
| Root config has no `projectId`         | `projectId` is a package-level concern              |
| Package configs inherit root IDs       | Copy, do not re-derive                              |
| Update `lastUpdated` on every write    | ISO 8601 timestamp                                  |
| Validate after writing                 | Re-read file and verify IDs match FlowState records |

## Quick Diagnostics

```bash
# Check root config
cat .flowstate/config.json | python3 -m json.tool

# Check a package config
cat packages/flowstate-cli/.flowstate/config.json | python3 -m json.tool

# Find all package configs in a monorepo
find packages -name "config.json" -path "*/.flowstate/*" | sort

# Find packages MISSING a config
for dir in packages/*/; do
  if [ ! -f "${dir}.flowstate/config.json" ]; then
    echo "MISSING: ${dir}"
  fi
done
```

## Error Reference

| Error                               | Likely Cause                               | Fix                                           |
| ----------------------------------- | ------------------------------------------ | --------------------------------------------- |
| `"orgId" missing`                   | Config never initialized                   | Run workspace-registration-process            |
| `collection-get orgs` → 404         | Org deleted or wrong ID                    | Ask user for correct orgId                    |
| `collection-get workspaces` → 404   | Workspace deleted                          | Re-register workspace                         |
| `collection-get projects` → 404     | Project deleted                            | Create with same ID from config               |
| Task created with wrong `projectId` | Used root config instead of package config | Always read nearest `.flowstate/config.json`  |
| Package has no `projectId`          | Audit never run                            | workspace-codebase-audit-process              |
| `"orgId": "default"` in config      | Placeholder used                           | Replace with real org ID, update all entities |
