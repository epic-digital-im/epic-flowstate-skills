# Org Schema

**Status:** Active
**Collection:** `orgs`
**ID Prefix:** `org_`
**Hierarchy Level:** 1 (root)
**Parent Required:** None — orgs are the root of all FlowState data

---

## Absolute Rule: IDs Are Never Fabricated

All FlowState IDs are auto-generated by MCP tools. Agents NEVER manually construct, guess, or infer IDs. The `orgId` must be read from `.flowstate/config.json`, never derived from a project name, repo name, or any other source. If the config file does not exist or `orgId` is missing, ask the user.

---

## Overview

An **org** (organization) is the multi-tenant root in FlowState. Every other entity (`workspaces`, `projects`, `milestones`, `tasks`, `codebases`, etc.) is scoped to an org via `orgId`. You cannot create any entity without a valid `orgId`.

The `orgId` is stored in `.flowstate/config.json` at the repository root and inherited by all child objects.

```
Org
└── Workspace
    └── Codebase
    └── Project
        └── Milestone
            └── Task
```

## Schema

### Required Fields

| Field       | Type    | Description                 | Example             |
| ----------- | ------- | --------------------------- | ------------------- |
| `id`        | string  | Auto-generated primary key  | `org_9f3omFEY2H`    |
| `name`      | string  | Kebab-case org identifier   | `epic-digital-im`   |
| `archived`  | boolean | Whether org is archived     | `false`             |
| `createdAt` | string  | ISO 8601 creation timestamp | `2025-01-01T00:00Z` |
| `updatedAt` | string  | ISO 8601 modified timestamp | `2025-01-01T00:00Z` |

### Optional Fields

| Field         | Type   | Description                 |
| ------------- | ------ | --------------------------- |
| `userId`      | string | Owner user ID               |
| `description` | string | Org description (max 5000)  |
| `metadata`    | object | Custom metadata (see below) |

### Metadata Fields (`OrgMetadata`)

| Field              | Type   | Description           |
| ------------------ | ------ | --------------------- |
| `colorSchemeIndex` | number | UI color scheme index |

### Collection Indexes

| Index       | Purpose            |
| ----------- | ------------------ |
| `createdAt` | Default sort order |

## Linking Rules

### Where `orgId` Is Stored

Every repository must have `orgId` in its root `.flowstate/config.json`:

```json
{
  "orgId": "org_9f3omFEY2H",
  "workspaceId": "...",
  "codebaseId": "..."
}
```

### Resolution Order

When an agent needs `orgId`:

1. Read `.flowstate/config.json` at the repository root
2. If not present, read the nearest parent directory `.flowstate/config.json`
3. If still not found → **error**: must create an org and initialize the config

### NEVER Use Placeholders

```json
// WRONG
{ "orgId": "default" }
{ "orgId": "org_default" }
{ "orgId": "flowstate" }

// CORRECT — always a real ID from FlowState
{ "orgId": "org_9f3omFEY2H" }
```

## Pre-Flight Check

Before creating any FlowState entity, verify the org exists:

```
collection-get orgs <orgId>
```

If this returns an error or empty result:

1. The org has been deleted or the ID is wrong
2. Ask the user which org to use
3. **Do not proceed** with entity creation using an invalid `orgId`

---

## Creating an Org

Create an org only when no org exists for the project. Orgs are typically created once per company/team.

```
collection-create orgs {
  name: "<org-name>",
  archived: false
}
```

After creation:

1. Record the generated `org_XXXXX` ID
2. Write it to `.flowstate/config.json` as `orgId`
3. Commit the config file

---

## Error Handling

| Situation                         | Action                                              |
| --------------------------------- | --------------------------------------------------- |
| No `.flowstate/config.json`       | Run workspace-registration-process first            |
| `orgId` missing from config       | Ask user for org ID or create a new org             |
| `collection-get orgs` returns 404 | Org deleted or ID wrong — ask user                  |
| Multiple orgs, unsure which       | List orgs with `collection-query orgs {}`, ask user |

## Conventions

| Item          | Convention                              |
| ------------- | --------------------------------------- |
| Org ID format | `org_XXXXX` (auto-generated)            |
| Org name      | kebab-case company or team name         |
| Config field  | `orgId` in `.flowstate/config.json`     |
| Scope         | All entities in FlowState require orgId |
